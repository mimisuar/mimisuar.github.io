Things I've done in my free time

SES (Sheep Entertainment System)

This is a small project I started working on back in a highschool. I was interesting in trying to write a really barebones NES emulator, but found it too challenging for me at the time so I started working on a fantasy console. I designed a small CPU with memory specifications and CPU flags. I needed someway to actually code the thing, so I wrote a little assembly-inspired language, and a compiler I wrote in Lua. The language, SASM (SES Assembly), had support for procedures, macros, inline memory initialization (.db), and a simple "#include" function.

The actually project was written in C++ using SFML for windowing and rendering. I was able to implement all of the opcodes from the "CPU", and I added basic input support, so you can use the keyboard as a controller. I also designed a sandboxed filesystem for the console, allowing programs to store and recall data. The last thing I was able to implement in the project befor graduating high school was a graphics system, allowing programs to render simple sprites to the screen.

Currently, the project is in a state of limbo, but I tinker with it from time to time. My current objectives are to fix the graphics system, and to implement a trivial audio synthesis module.

Legendary Bard

During my junior year of high school, I took part in the Maine App Challenge. Over the course of several months, I designed and built a game for teaching relative pitch (being able to recognize intervals instead of the pitches themselves). As a musician, I've found that the intervals between notes tends to be more significant than the notes themselves. In the game, the monsters will play a series of tones (the monsters are associated with an organ sound) which the player responds to using a piano. If the player gets it right, they get to attack the monster.

The project was built using Java and LibGDX, a fairly low-level graphics/audio/input API, using Android Studio. The game earned me a spot as a finalist in the competition.

Because I made this while I was still in high school, I made some fairly poor design decisions (both code structure and game design), but I'm still proud of what I was able to accomplish by myself.
